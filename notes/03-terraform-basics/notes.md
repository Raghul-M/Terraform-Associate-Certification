## Standard Project Structure

```
project/
├── main.tf              # Primary resource definitions
├── variables.tf         # Input variable declarations
├── outputs.tf           # Output value declarations
├── terraform.tfvars     # Variable values (auto-loaded)
├── providers.tf         # Provider configurations
├── versions.tf          # Version constraints
├── .terraform/          # Downloaded providers/modules (DO NOT commit)
├── .terraform.lock.hcl  # Dependency lock file (COMMIT to VCS)
└── terraform.tfstate    # State file (NEVER commit)
```

---

## What to Commit vs Ignore

| File/Folder | Commit to Git? |
|-------------|----------------|
| `*.tf` files | ✅ Yes |
| `.terraform.lock.hcl` | ✅ Yes |
| `terraform.tfvars` | ⚠️ Only if no secrets |
| `.terraform/` | ❌ No (auto-generated) |
| `terraform.tfstate` | ❌ Never (contains secrets) |
| `*.tfplan` | ❌ No |

## HCL Basics

HCL = **HashiCorp Configuration Language**. Human-readable, also supports `.tf.json`.

```hcl
# Block syntax
block_type "label1" "label2" {
  argument = "value"
  nested_block {
    key = "value"
  }
}
```

---

## The `terraform {}` Block

```hcl
terraform {
  required_version = ">= 1.0"

  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
}
```

---

## Version Constraint Operators

| Operator | Meaning | Example |
|----------|---------|---------|
| `=` | Exactly this version | `= 1.0.0` |
| `>=` | This version or newer | `>= 1.0` |
| `~>` | Pessimistic constraint (most common) | `~> 5.0` means >= 5.0, < 6.0 |
| `>= X, < Y` | Range | `>= 1.0, < 2.0` |

**Exam favorite:** `~> 5.0` allows `5.0`, `5.1`, `5.99` but NOT `6.0`.

## Provider Configuration

```hcl
provider "aws" {
  region = "us-east-1"
}

# Multiple providers (multi-region)
provider "aws" {
  alias  = "west"
  region = "us-west-2"
}

resource "aws_instance" "web_west" {
  provider = aws.west
  # ...
}
```

## `.terraform/` vs `.terraform.lock.hcl`

| | `.terraform/` | `.terraform.lock.hcl` |
|--|---------------|----------------------|
| Contains | Downloaded provider binaries, modules | Exact provider version checksums |
| Generated by | `terraform init` | `terraform init` |
| Commit? | No | **Yes** |
| Purpose | Local plugin cache | Reproducible builds across team/CI |

---

## Provider Architecture

1. You declare `required_providers` with version constraints
2. `terraform init` downloads matching provider plugins
3. Providers are **separate binaries** that talk to cloud APIs
4. Lock file pins exact versions for consistency

## Useful Commands

```bash
terraform version              # Check CLI version
terraform -help                # List commands
terraform <command> -help      # Help for specific command
terraform providers            # List required providers
```

## Exam Traps

| Trap | Truth |
|------|-------|
| `.terraform/` should be committed | **No** — it's a local cache |
| Lock file is optional | **No** — commit it for reproducibility |
| `terraform.tfstate` is safe in Git | **No** — contains plaintext secrets |
| `~> 1.0` allows version 2.0 | **No** — allows >= 1.0, < 2.0 |
| Provider stores state | **No** — backend stores state |
